Basic steps for development on github

Posted on 2019-04-04

Followed my friend ryan’s introduction on how to build a blog on github, actually I meant to start from zero as I know some basic usage of html, css and js, but since hexo is highly recommened to make life easier, then just give it a try.

Basicly is following https://ryanluoxu.github.io/2017/11/24/%E7%94%A8-Hexo-%E5%92%8C-GitHub-Pages-%E6%90%AD%E5%BB%BA%E5%8D%9A%E5%AE%A2/

But since I’m using vs code on windows, here is the point view of set up

Prepare the environment

  • A terminal
    https://code.visualstudio.com/
    VS code is a powerful tool for code editing and has many functions, it even support java.
    VS code has build-in terminal (ctrl+`), which is good enough to use
  • Git
    get it from https://git-scm.com/, git mesh is not necessary since we can use the terminal in VS code
  • Node.js
    get it from https://nodejs.org/en/
  • Install hexo
    After installation of git and node.js, restart vs code, ctrl + ` to call out the terminal
    Make use of the basic command ‘cd’ to change the current folder to whatever you like
    type ‘npm install hexo-cli -g’ to install the hexo framework, if shows erro, something wrong with your node.js
    type ‘npm install hexo-deployer-git –save’ to install hexo-deployer-git, this is needed for later deployment
    type ‘hexo init blog’ and wait, this will setup the environment for you local hexo blog
    type ‘cd blog’ to change the current folder to the hexo blog folder
    type ‘hexo server’ and your shall be able to visit the local blog at http://localhost:4000/
    you may wish to post your own blog by type ‘hexo new “blog name”‘, and the blog will be at blog/source/_posts/blog-name.md

Prepare the account

  • All your need is a github account, https://github.com/, go and get one.
  • Create a repository named username.github.io, username is your perferred username

Deploying

  • Open the _config.yml file under blog folder, you can use vs code to edit this file
  • Under #Deployment section
    deploy:
    type: git   
repo: https://github.com/username/username.github.io
branch: master
  • Save the file, come back to the terminal
    type ‘hexo clean’ to clean the folder
    type ‘hexo deploy’ to deploy to github, if shows erro, check your config file, take care of the repo link
    You shall see a pop up window from github and asking for your log-in
    After log-in, the deployment will carry on until you saw some message like ?[32mINFO ?[39m Deploy done: ?[35mgit?[39m

Check your own blog

  • Go to your github to confirm there is some files uploaded in your repository
  • Your blog will be at username.github.io

Other configuration

  • Configuration for other parts can be followed at this document https://hexo.io/docs/configuration
  • After configuration and other edit, you can alway check it locally first by ‘hexo server’
  • If happy with the result, ‘hexo clean’ && ‘hexo deploy’

Enjoy!
