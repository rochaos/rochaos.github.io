Tips

  • all statemtns in Java must end with a semi-colon;
  • for if and else, {} is unnecessary if action is only one line;
  • parameters are varibables in a method definition, and arguments are the data to pass into the parameters

Baisc Structure (from my current understanding)

Variables and Operators

8 Basic data types (predefined) / Primitive data types

  • For storing integers
    • byte (-128 to 127), use 1 byte (8bits) of storage space.
    • short (-32768 to 32767), use 2 byte (16bits) of storage space.
    • int (-2^31 to 2^31-1), use 4 byte (32bits) of storage space, most commonly used data type
    • long (-2^63 to 2^63-1), use 8 byte (64bits) of storage space, need to add suffix ‘L’ to end of the number, rarely used.
  • For storing floating point number
    • float (-3.40282347E38 to 3.40282347E38), use 4 byte (32bits) of storage space, as precision of about 7 digits (1.2345678), need to add suffix ‘F’ to the end of number.
    • double (-1.79769……E308 to 1.79769….E308), use 8 byte (64bits) of storage space, with precision of about 15 digits, by default Jave treat floating point number as double.

  • Other primitive data type

    • char (single unicode characters), use 2 bytes (16bits) of memory, enclose in single quotes, eg. char grade = ‘A’;
    • boolean (true and false), require only 1 bit of memory, but the size depends on the smallest chunk of memory

  • Other types are reference types, only store the address, the size depends on the addressing mechanism of the machine (32bit system-4bytes, 64bit system-8 bytes)

Basic Operators

  • x = x + y is equal to x +=y
  • For division, int / int = round down int, any of non-int will give a truncated(shortened) non-int
    ( 7/2=3, 7.0 /2 = 3.5, 7/2.0=3.5, 7.0/2.0=3.5)
  • System.out.println(counter++) means print counter first, then counter++
    System.out.println(++counter) means counter++ first, then print counter

Type Casting

  • Widening primitive conversion, smaller data to larger type, can be done directly

  • Narrowing primitive conversion, larger to smaller, need use a pair of parenthesis(), may lost data, decimal portion is truncated.

Object Oriented Programming I

Writing our own class

An example is public class ManagementStaff{}
Public class can be accessed by any class in the program
By default the access level is package-private
only one public class per java file and the public class must have the same name as the file
For another public class, we need to create another file
We can call other classes in the main(){}

Fields

Fields are inside the {} of class, used to store data

A field can be declared as private, public or protected.
private file can only be accessed from within the Class itself

Methods

  • A method is code block that performs a certain task
    AccessMofifier returntype nameOfMethod(parameters){}
    In the case that the method does not return any value, returntype will be void.
    (parameters) can be empty, ()when no parametes are needed.
  • A method can access all the fields that are declared inside the class.
  • A method can declare local variables which only avalibale within the method.

  • A return keyword is used to exit the method with returntype of data.

  • Overloading
    Create two methods of the same name with different signatures
    e.g. public int calculatePay() and public int calculatePay(int bonus, int allowance)

  • Getter and Setter Methods
    To allow other classes to access the private fields

    • Setter method, normally use the setNameOfField
      Set the value of a private field

    • Getter method, normally use the getNameOfField
      Get the value of a private field

Constructors

  • A constructor is a block of code that construct an object from the class template
  • Always has the same name as the class
  • Commonly used to initialize the fields of the class
  • Does not return any value but no need to use void keyword when declaring
  • Support overloading
  • If not declared, default constructor will be generated, initialize fileds with 0 or null
  • A typical constructor is public ClassName(parameters){}

Instantiating an Object

  • An object is also known as an instance.
  • Instantiate a object in the main() method inside the class

Static

A keyword used call methods or access fields in the Class without creating a object

  • Can be directly called by MyClass.data and MyClass.doSomething
  • Static methods cannot be overridden, but can be hidden by subclass
    tips. Array class is declared as static, so can call methods like Arrays.sort() directly

Advanced Methods

Useing Arrays in Method (reference data type)

  • Use array as parameters
    public void thisAcceptArray(int[] a){}
  • Use array as return type

Passing Primitive vs Reference Type Parameters

  • For primitive type variables
    • Change of value in the method will be invalid after the method ends.
    • Only value is passed, no modify to data at origin address
  • For reference type variables
    • Change of value in the method will change the origin data.
    • The address is passed, so modify will last

Object-Oriented Programming II

Inheritance

Basic Inherit

  • Create a new class from an existing class so we can reuse existing code
  • Parent / base classes (superclasses) derive child classes (subclasses)

  • To declare, use the keyword extends

  • extends is unnecessary when inheriting from the Object class

  • child class does not inherit the private fileds and methods
  • Java will auto call a parameterless constructor
  • Use keyword super(parameters) to call a non parameterless constructor and pass the value to it
  • To call a method in the parent class, use super.method()
  • Constructors also can access the value of private fileds

Overriding a method

  • writing a new version of the method in the child class
  • Use keyword @Override before the method

Annotations

  • The line above method with @ is known as annotation
  • @Override is optional for overriding, but can prevent compilation errors
  • Annotations are case sensitive, fill list of predefined annotaions
    https://docs.oracle.com/javase/tutorial/java/annotations/predefined.html

Polymorphism

  • Ability to use the correct method for an object based on its run-time type
  • Example

Special Classes and Methods

Abstract Classes and Methods

  • A special type of parent class
  • Strickly to be a parent class and cannnot be instantiated
  • Abstract methods have no body and can only exist in abstract classes
  • Abstract methods must be implemented in the child class

Interfaces

  • Also cannot be instantiated
  • Must be implemented by classes or extended by other interfaces
  • Can only contain abstract methods(no body methods) and constants(final fields) [JAVA 7]
  • All methods in interface are implicitly public
  • All constant values are implicity public, static and final
  • A class can only extend one abstract class but can implement multiple inferfaces

  • A class that implements an interface must implement all the abstract methods in the interface

  • Keyword implements is used to indicate the interface

  • static and default methods are added since JAVA 8, cannot be override
  • default methods of an interface can be inherited
  • static methods of an interface cannot be inherited

Collections

A type of framework provided by Java to easily store and manipulate objects

The Java Collections Framework

  • A set of pre-written classes and interfaces
  • Standardizs the way in which groups of objects are handled
  • Can only store objects

    Autoboxing and Unboxing

  • The 8 primitive types are not objects, so ofentimes need to be convert into an object
  • Wrapper classes are provided
Primitive data boolean char byte short int long float double
Wrapper class Boolean Character Byte Short Integer Long Float Double
  • Wrapper class is reference type data
  • To convert primitive data into object
    Integer intObject = new Integer(100) //create a new object
    // method 1, most accept primitive and string, char only accept primitive, float accept double
    Integer intObject = Integer.valueOf(100) //may return a cached object, more efficient than constructors
    //method 2, most accept primitive and string, char only accept primitive, float DOESN’T accept double
    Double douObj = Double.valueOf(int) is OK as widening conversion is automatically done by compiler
    Short shortObj = Short.valueOf(int) is NOT OK as implicit narrowing is not automatically for method calls
  • To conver object back to primitive data
    int m = intObject.intValue()
  • With autoboxing and unboxing, covert can be simplified as
    Integer intObject = 100 and int m = intObject
  • To conver object to primitive type
    int n = Integer.parseInt("5") or double q = Double.parseDouble("5.1")

List

Similar to array, but the size can be changed

ArrayList

  • Can only store objects
  • Need to import java.util.ArrayList before use
  • Add or delete of element will shift other elements, may cause delay for large ArrayList
  • Declaring and instantiating

ArrayList Methods

The following methods are pre-written, typical usage will be nameOfArrayList.method()

  • .add(data), to add members to a list
    • .add(index, data) to add data at specific position
  • .set(index, data) to replace data at specific position
  • .remove(index) to remove data at specific position
  • .get(index) to get data at specific position
  • .size() to get the number of elements in the list
  • .contains(data) to check if list contains the data, will return true / false
  • .indexOf(data) to get the index of the first occurrence of an element
    will return -1 if element does not exist

  • .toArray() returns an array of object containing all the elements, from first to last element

  • .clear() to remove all items in a list

  • Complete list of all the ArrayList methods at https://docs.oracle.com/javase/8/docs/api/java/util/ArrayList.html

LinkedList

  • A LinkedList will sotre the neighbouring addresses of the element
    null-Data-nextAddress - lastAddress-Data-nextAddress - lastAddress-Data-null
  • No shift of data, it only update the linked addresses
  • Java will auto delete the not pointed element
  • Need to import java.util.LinkedList before use
  • Implements the Queue and Deque interfaces on top of the List interface
  • Needs more memory and takes longer time for search
  • Declaring and instantiating

LinkedList Methods

Share most methods of ArrayList Methods, but have queue and deque interfaces with following methods

  • .pull() return the first element (head) and remove it from the list
  • .peek() return the first element, will return null for empty list
  • .getFirst() return the first element, will give NoSuchElementException for empty list
  • .getLast() return the last element, will give NoSuchElementExcepetion for empty list
  • Complete list of all LinkedList methods at https://docs.oracle.com/javase/8/docs/api/java/util/LinkedList.html

File Handling

Use import java.io.* to import entire java.io package, contains all the classes needed

Reading a Text File

  • FileReader reads file as stream of characters, reading in one character at a time
  • Usually wrap a BufferedReader object around the FileReader to improve the efficient
    BufferedReader reader = new BufferedReader (new FileReader("filename"))
  • To read a line, use String line and line = reader.readLine()
  • To release the resources after read, use reader.close()
  • try-with-resources statement (JAVA 7 and beyond), Java will auto close the object

Writing to a Text File

  • Use BufferedWriter and FileWriter class

Renaming and Deleting Files

  • Need to create the File objects first

  • file1.renameTo(file2), return true for successfully rename

  • file1.delete(), return true for successfully delete
  • If use try-catch, should catch

Advanced Java Topics

Generics

General work

  • Create a single class, interface or method that auto works with different types of data
    • Use type parameter during declare, class MyGenericsClass<T> {}
    • For interface, it will be interface MyInterface<T> {}
    • T will also work with any reference type and only reference type
  • Also allows for type-checking
    • Specify data type when instatiate an object, MyGenericClass<DataType> g = new MyGenericClass()

Bounded Types

  • Specify the type parameter as <T extends A> will only accept data types that are subtypes of A
    • <T extends Number> will only accept numeric data, include Integer and Double

Functional Interfaces and Lambda Expressions

  • An interface that contains one and only one non-implemented method

  • Can contain other static and default method

  • Can add @FunctionalInterface annotation
  • Can use lambda expressions to implement the functional interface
